# LangChain 自学建议

基于项目代码分析和模拟面试表现整理。

---

## 1. 当前学习模式

```
看文档 → 写 demo → python app/xxx.py 看结果
```

**优点：** 动手实践，每个知识点都跑了可执行脚本。

**问题：**
- 代码跑通了，但原理理解不深（batch vs batch_as_completed、stream 消息类型）
- 注释读到了，但没转化成自己的知识（项目里大量 ⚠️ 注释，面试时答不上来）
- 知识点是散的，没有连成体系

---

## 2. 四个改进建议

### ① 跑完后关掉文件，给自己讲一遍

每跑完一个 demo，回答三个问题：

| 问题 | 示例 |
|------|------|
| 这个功能是干什么的？ | stream_events 是逐步推送 state |
| 和相似功能的区别？ | invoke 等全部完成，stream 每步都推 |
| 有什么坑？ | version=v3 是实验性，会打 BetaWarning |

讲不出来 = 没真懂。

### ② 追查项目里的已知坑

项目中大量 `⚠️` 注释，针对每个坑去官网查**为什么会有这个坑**：

| 坑 | 涉及文件 |
|------|---------|
| `enable_thinking=True` 和 `tool_choice` 冲突 | `deepseek_model.py`、`agent_structured_output.py` |
| DashScope 的 `reasoning_content` 被 OpenAI 适配器丢弃 | `deepseek_model.py`、`thinking_model.py` |
| `create_deep_agent` 注入内置工具导致 token 飙升 | `agent_structured_output.py` |
| context 传了但不走 tool 就无效 | `agent_context.py` |

### ③ 从写 demo 进阶到改 demo

把两个功能组合起来：

- `agent_context.py` 的 context 注入 + `agent_stream.py` 的流式输出
- `agent_token_usage.py` 的 UsageMetadataCallbackHandler 加到其他 demo

这是从"会写"到"会用"的跨越。

### ④ 补充笔记

现有 `docs/` 下有架构文档，方向很好。建议补充：

- **对比笔记** — 容易混淆的概念：create_agent vs create_deep_agent、invoke vs stream、batch vs batch_as_completed
- **坑点笔记** — 实际踩过的坑和解决方案

## 3. 学习资源优先级

| 优先级 | 内容 | 原因 |
|--------|------|------|
| 🔴 高 | LangChain 官网 Tutorials | 系统化，比零散看 API 文档好 |
| 🔴 高 | LangGraph 官方文档 | 项目大部分功能依赖 LangGraph（checkpointer、stream_events） |
| 🟡 中 | Conceptual Guide | 理解设计理念（Agent vs Chain） |
| 🟢 低 | API Reference | 当字典查，不需要通读 |

---

## 4. 检验标准

不看"看了多少页文档"，用这个衡量：

- 别人问区别，你能**不看代码讲清楚**
- 写新功能，你知道**该用哪个 API，有什么限制**
- 出错了，你能**猜到大概是哪里的问题**
